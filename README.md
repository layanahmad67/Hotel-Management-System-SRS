Hotel Management System – Software Requirements Specification

Overview

This project was developed as part of the Software Requirements Engineering (SW210) course. It presents the Software Requirements Specification (SRS) for a Hotel Management System designed to streamline hotel operations, improve customer experience, and support efficient reservation and management processes.

Project Objectives

Analyze business and user requirements.
Define the system scope and stakeholders.
Document functional and non-functional requirements.
Produce a structured Software Requirements Specification (SRS).

System Features

User registration and authentication.
Hotel room reservation and management.
Check-in and check-out management.
Online and manual payment support.
Event hall booking.
Food and beverage reservation.
Report generation for reservations and payments.

Non-Functional Requirements

Usability
Security
Reliability
Accuracy
Maintainability
